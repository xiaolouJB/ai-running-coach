# 贡献指南

感谢你对 AI Running Coach 的关注！本文件说明如何为项目贡献适配器、训练理论和其他内容。

---

## 一、适配器贡献规范

适配器让 AI 教练核心逻辑能与不同品牌手表配合工作。如果你熟悉某款手表的 MCP 接入方式，欢迎贡献适配器。

### 1.1 前提条件

- 该设备有可用的 MCP 服务端点（或可以通过现有 API 包装成 MCP）
- 你已验证至少以下 4 个核心接口可用：`GET_SPORT_RECORDS`、`GET_RESTING_HR`、`GET_RECOVERY_STATUS`、`GET_FITNESS_OVERVIEW`
- 接口返回的数据字段与 ADAPTERS.md 中的字段说明可以对应

### 1.2 适配器格式

在 ADAPTERS.md 末尾追加以下格式的新适配器章节：

```markdown
## N、[设备品牌名称]适配器

**状态**：🚧 开发中 / ✅ 正式支持
**MCP 端点**：[MCP 服务 URL]
**认证方式**：[OAuth 2.0 / API Key / 其他]
**支持的设备型号**：[例如：Forerunner 系列、Fenix 系列]

### N.1 工具映射表

| 抽象接口 | [设备] MCP 工具名 | 关键参数 | 说明 |
|---------|----------------|---------|------|
| `GET_SPORT_RECORDS` | `mcp__[设备]__[工具名]` | `startDate`, `endDate` | 说明 |
| `GET_RESTING_HR` | `mcp__[设备]__[工具名]` | `startDate`, `endDate` | 说明 |
...

### N.2 字段差异说明

如果某设备的字段名或单位与 COROS 基准不同，在此说明归一化方式：

| 差异项 | [设备]原始值 | 归一化为 | 处理方式 |
|--------|-----------|--------|---------|
| [例如配速单位] | min/mile | sec/km | × 37.28 |
```

### 1.3 测试要求

提交适配器前，请验证：
- [ ] 能成功拉取最近 7 天的运动记录
- [ ] VO2max / 体能评估字段返回非空值
- [ ] 恢复状态字段返回合理数值
- [ ] MCP 连接说明在 USER_GUIDE.md 的安装步骤基础上可以复用

## 二、训练理论贡献规范

THEORY_LIBRARY.md 第四章是开放的用户自定义区，任何人可以追加训练理论笔记。如果你想将理论内容提交到主库（第一至三章），需要满足以下条件。

### 2.1 理论内容要求

- **有明确出处**：需引用具体书籍、论文或被广泛认可的教练体系
- **有 AI 使用规则**：需将理论转化为 AI 可执行的判断条件（格式参见第一章 1.3 节）
- **不与现有理论矛盾**：如有冲突，需在文中说明处理优先级

### 2.2 格式模板

```markdown
## 第N章：[理论名称]

**出处**：[作者], *[书名/论文名]* ([年份]), [出版社].
**核心贡献**：[一句话说明该理论的核心贡献]

### N.1 [理论要点标题]

[理论内容说明]

### N.2 AI 使用规则

- [条件] → [AI 执行的动作]
- [条件] → [AI 执行的动作]
```

## 三、问题反馈

如果你遇到以下问题，欢迎反馈：
- AI 生成的配速区间与你实际感受严重不符
- 某个 MCP 工具调用持续失败
- 课表逻辑有明显错误（如恢复日被安排了高强度训练）

反馈时请附上：
1. 你的设备型号和 COROS App 版本
2. 具体的报错信息或 AI 输出内容
3. 你的期望行为
