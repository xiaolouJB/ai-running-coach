## 二、用户说明区重复内容（原 SKILL.md 第一章）

> 移出原因：与 USER_GUIDE.md 内容高度重叠，USER_GUIDE.md 表达更清晰。

### 适合谁使用

| 用户画像 | 典型需求 |
|---------|---------|
| 🏅 有跑步基础、想冲击 PB 的跑者 | 系统化备赛，告别盲目刷量 |
| 📅 时间有限、想高效训练的上班族 | 每周 3-4 天，最大化训练收益 |
| 😵 看不懂训练计划、不知道配速区间的新手 | 全自动生成，中文解读，无需专业知识 |
| 📊 有运动手表但不知道怎么看数据的用户 | AI 帮你读数据、给建议 |

### 解决的核心问题

| 传统痛点 | 本 Skill 的解法 |
|---------|----------------|
| 网上的计划千篇一律，不适合自己 | 基于你的真实数据自动生成专属计划 |
| 需要手动填写复杂的体能测试表格 | 自动从运动手表接口拉取，零填写 |
| 训练完了不知道跑得好不好 | 自动打分 + 理论复盘，每次训练有评语 |
| 计划是死的，跑得好/不好都不会调整 | 每周动态更新，能力提升则加量，疲劳则减量 |
| 看不懂配速区间、心率区间 | 用你的数据自动推算，用白话解释 |

### 核心价值段落

```
传统计划：制定 → 执行（计划是死的）
AI 教练：制定 → 执行 → 自动分析 → 动态调整 → 再执行（计划会进化）
```
