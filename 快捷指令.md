# ⚡ AI 跑步教练 · 快捷指令模板

> **使用方法**：根据你的场景选择对应模板，将 `[ ]` 内容替换为你的实际信息，复制发送给 Claude 即可。  
> 无需记忆任何命令——像和真人教练说话一样输入即可。

---

## 场景一：初始化（第一次使用）

**何时用**：第一次启动 AI 教练，生成专属训练计划时。

```
请帮我初始化 AI 跑步教练系统，读取我高驰手表最近6周的训练数据，为我生成专属课表。

我的信息：
- 目标比赛：[半程马拉松 / 全程马拉松 / 10公里 / 5公里]
- 目标完赛时间：[例如：1小时30分]
- 每周可训练天数：[例如：4天]
- 目标比赛日期：[例如：2026年11月15日]
- 每周跑量上限：[例如：60公里，或不限]
- 历史最佳成绩（没有可不填，AI会从手表推算）：[例如：半马1:38]
```

---

## 场景二：每周更新（日常最常用）

**何时用**：每周训练结束后（推荐每周日晚上），更新本周数据并获取下周计划。

```
更新本周训练数据
```

就这一句话。Claude 会自动：
- 从高驰手表拉取本周所有跑步记录
- 给每次训练打分、写评语
- 分析本周整体训练质量
- 根据实际表现调整下周课表

**如果本周有特殊情况，可以附加说明：**

```
更新本周训练数据。
备注：[例如：周三因为下雨跳过了训练 / 周五膝盖有点不舒服跑得保守了 / 这周出差只跑了3次]
```

---

## 场景三：赛前复盘（赛前 2-4 周）

**何时用**：比赛临近时，评估当前状态和制定赛前减量策略。

```
请帮我做赛前状态评估。

比赛信息：
- 比赛项目：[半程马拉松 / 全程马拉松]
- 比赛日期：[例如：2026年11月15日]
- 目标时间：[例如：1小时30分]

请分析我目前的训练状态，预测我的比赛成绩，并制定赛前减量（Taper）计划。
```

## 场景四：随时复盘（按需使用）

**何时用**：想深入了解某次训练、本周整体状态、或近期训练趋势时。无固定时间，随时可用。

### 4A · 单次训练复盘

```
帮我分析一下[今天/昨天/最近那次]的跑步
```

AI 会分析：配速稳定性、心率漂移、步频、训练负荷、与课表目标的偏差，并在最后询问你的主观感受（1-10分）。

### 4B · 本周复盘

```
复盘一下这周的训练
```

AI 会综合本次更新的所有数据 + 本自然周全部记录，分析：类型完成情况、强度分布（是否符合 80/20）、恢复指标、VDOT 趋势，以及是否需要调整下周计划。

### 4C · 近期状态复盘（自定义天数）

```
帮我看看我最近[14天/一个月/N天]的训练状态
```

AI 会分析：长期训练负荷曲线、HRV 趋势、睡眠质量、疲劳积累评估，以及距离目标的进展情况。

> **提示**：复盘结束时 AI 会询问主观感受评分（1-10分），这个信息会保存到课表中，帮助 AI 更准确评估训练质量。如果不想被问，告诉 AI"不用问感受了"即可。

## 临时查询（随时可用）

无需特定格式，直接用自然语言问即可：

| 想了解的内容 | 说法示例 |
|------------|---------|
| 今天适合训练吗 | `今天能不能跑？` |
| 最近恢复状态 | `我现在恢复得怎么样？` |
| 预测比赛成绩 | `按我现在的状态，比赛能跑多少？` |
| 睡眠和 HRV | `我最近睡眠质量怎么样？` |
| 心率趋势 | `我最近静息心率有变化吗？` |
| 训练强度检查 | `我最近是不是练得太狠了？` |
| 查看课表变更 | `这周课表有没有调整，为什么改的？` |

## 场景五：生成本周打卡图（用于分享）

**何时用**：每周更新数据后，想生成一张可发小红书 / 朋友圈的周复盘打卡图时。  
**前提**：必须先执行**场景二（更新本周训练数据）**，AI 才有本周课表记录可读取。

```
生成本周打卡图
```

就这一句话。AI 会自动：
- 从课表文件读取本周训练记录（含每日达成率、力量日安排）
- 从高驰手表拉取 VDOT 实测值、静息心率、HRV、恢复状态
- 生成 1080×1440 小红书规格 HTML 文件，保存到 `schedules/` 文件夹
- 告知你文件路径，用浏览器打开后点「下载 PNG」即可保存

**打卡图包含以下内容**：
- 5 项核心数据：训练次数 / 总跑量 / 均心率 / VDOT / 恢复状态
- 本周课表实录（跑步行 + 力量行 + 达成率）
- 日跑量 7 天柱状图
- AI 复盘 3 条洞察（正向 · 警示 · 亮点）
- 下周迭代计划 3 张卡
- 下周课表预览（含力量日）
- 项目 GitHub 二维码
